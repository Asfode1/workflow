# Parcel Tracker

Система отслеживания посылок с использованием Go и SQLite.

## Описание

Приложение позволяет:
- Регистрировать новые посылки
- Отслеживать статус посылок (зарегистрирована, отправлена, доставлена)
- Изменять адрес посылки (только для зарегистрированных)
- Удалять посылки (только зарегистрированные)
- Просматривать все посылки клиента

## Локальная разработка

### Требования
- Go 1.21+
- SQLite

### Установка и запуск

1. Клонируйте репозиторий:
```bash
git clone <repository-url>
cd final-main
```

2. Установите зависимости:
```bash
go mod download
```

3. Запустите приложение:
```bash
go run .
```

4. Запустите тесты:
```bash
go test -v ./...
```

## Docker

### Сборка образа
```bash
docker build -t parcel-tracker .
```

### Запуск контейнера
```bash
docker run --rm parcel-tracker
```

## GitHub Actions

Проект настроен с двумя workflow:

1. **Test and Lint** - запускается при push и PR:
   - Проверка кода линтером
   - Запуск тестов
   - Сборка приложения

2. **Docker Publish** - запускается при создании тегов:
   - Сборка Docker образа
   - Публикация в DockerHub

## Настройка секретов

Для работы с DockerHub необходимо настроить следующие секреты в GitHub:

- `DOCKERHUB_USERNAME` - имя пользователя DockerHub
- `DOCKERHUB_TOKEN` - токен доступа DockerHub

## Структура проекта

- `main.go` - основная логика приложения
- `parcel.go` - работа с базой данных
- `parcel_test.go` - тесты
- `Dockerfile` - конфигурация Docker
- `.github/workflows/` - GitHub Actions workflows
